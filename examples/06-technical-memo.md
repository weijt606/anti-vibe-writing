# 示例 6：内部技术备忘

场景预设：专业报告 / 内部备忘。模式：默认。

## 前

```md
## 【背景】

目前的数据管道在一定程度上能够满足业务需求。

## 【现状分析】

- 系统整体运行较为稳定
- 但仍存在一定的优化空间
- 未来或许可以进一步提升

## 【建议】

综合考虑各方面因素，建议采取较为均衡的策略，
具体问题具体分析，因时制宜，因事制宜。
```

## 后

```md
现在的数据管道还能用，但有两个已知风险：上游断流没有兜底、重试逻辑
在并发高的时候会重复写库。两个都还没出过线上事故。

如果这季度不动，省两周工，继续承担每月一两次脏数据的清理成本。
如果现在动，多两周排期，之后这两类问题不用再回头处理。

我倾向现在动。但如果 Q2 排期已经满，可以接受拖到 Q3。
```

## 改了什么

- 段头【背景】【现状】【建议】全删
- "一定程度上 / 较为稳定 / 一定的优化空间 / 或许 / 可能"几乎全部具体化
- "综合考虑各方面因素"换成具体的取舍：动 vs 不动各自的代价
- "具体问题具体分析"换成真正的分情况：什么情况下选哪个
- 结尾下了倾向（"我倾向现在动"），但保留了让步条件——这才是真平衡

## 为什么默认模式

技术备忘需要权威感，不需要博客的随意。但权威感不是"用更多套话"——
是"明确下判断、量化不确定性、写出取舍"。默认干净模式就够了，不需要
注入语气词或残句。
