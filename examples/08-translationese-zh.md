# 示例 8：中文翻译腔专项（1.3.0 新 pattern）

这条专门演示 1.3.0 新加的中文痕迹：用身体动作写抽象、形容词+冒号预判读者、抽象名词当主语、回避系动词"是"、同义词循环、四字成语堆砌、欧化句式。

场景：技术博客里的一段。模式：默认（干净）。

## 前

```
作为一个长期关注性能的团队，我们深知缓存的重要性。这套系统堪称我们架构
演进中的里程碑：它的设计逻辑非常清晰——通过对请求进行拦截，从而接住高
并发流量，避免数据库被瞬间击穿。

工程上的现实比预想的更复杂。该方案在落地过程中，一方面需要收口各类边界
情况，另一方面也要扛住持续的压力测试。所幸这套机制最终得心应手、游刃
有余，可谓事半功倍。
```

## 后

```
我们在数据库前面加了一层缓存。请求先过缓存，命中就直接返回，不命中才查库。
上线后高峰期的数据库 QPS 从 8000 降到 1200。

真做起来比想的麻烦：缓存穿透、击穿、雪崩三种情况都得单独处理，压测连着
跑了两周才稳住。现在没再出过事故。
```

## 改了什么（一条 pattern 对一处修复）

- **作为一个……（翻译腔开头）**：删掉，直接说我们做了什么
- **堪称 / 可谓（回避"是"）**："堪称里程碑""可谓事半功倍"全删，没有这层抬调子的话
- **形容词 + 冒号预判读者**："它的设计逻辑非常清晰："删掉，让"先过缓存、命中返回"这个事实自己说话
- **用身体动作写抽象**："接住流量""收口边界情况""扛住压测"换成"过缓存""单独处理""跑了两周才稳住"
- **击穿**：当动词用的"被瞬间击穿"删掉；但"缓存击穿"是行话，和"穿透、雪崩"并列时保留，不是见词就杀，看它是不是真术语
- **抽象名词当主语**："工程上的现实比预想的更复杂"换成具体的"真做起来比想的麻烦"
- **欧化排比**："一方面……另一方面……""通过……从而……"全部拆成动作链
- **四字成语堆砌**："得心应手、游刃有余、事半功倍"三连删干净
- **同义词循环**："这套系统 / 该方案 / 这套机制"指的是一个东西，后版只说"缓存"

## 效果

前版五个四字成语、三处身体动作词、零个具体数字。后版没有一个成语，多了两个可验证的数字（QPS 8000→1200、压测两周），读出来像工程师在讲自己干的活，不像产品稿。
