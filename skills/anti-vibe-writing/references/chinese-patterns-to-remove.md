# 中文 AI 味：要删的痕迹

这份参考专门记录中文文档里的生成感。英文那套规则覆盖不到这些，中文有自己一套套话、句式和结构惯性。

按"词汇、句式、结构、语气"四层组织。命中任何一项，基本可以确认是 AI 起的稿。

## 词汇层：套话词

读到这些词，第一反应应该是问："不用这个词，原来要表达什么？"往往写得出更具体的版本。

互联网黑话：
- 赋能、打通、闭环、抓手、对齐、复盘、心智、链路、底层逻辑、生态位
- 一站式、全方位、全链路、端到端、全场景
- 颗粒度、上下文、落地（被滥用时）

虚动词：
- 助力 / 致力于 / 旨在 / 通过……来……
- 实现 / 完成 / 进行了……（把动作名词化再用"进行"重新激活）
- 具备……能力 / 拥有……特性
- 即可 / 便可 / 仅需

填充词：
- 在很大程度上 / 一定程度上 / 某种意义上
- 进一步 / 不断地 / 持续地
- 更加 / 愈发 / 越来越
- 切实 / 真正 / 实实在在地

夸饰词：
- 干货满满、硬核、深度好文、强烈推荐
- 革命性、颠覆性、划时代、全新升级
- 极致、完美、丝滑、无缝

四字成语堆砌：
- 一句里连缀两个以上的四字词，是 AI 中文的招牌之一。
- 高频的："一目了然、得心应手、事半功倍、不可或缺、至关重要、井井有条、游刃有余、相辅相成、息息相关"
- 弱：本工具功能强大、操作简便、界面美观，让你事半功倍、得心应手。
- 强：装上就能用，常用的功能在第一屏，不用翻菜单。

成语不是不能用，是 AI 用得太密、太顺。一句最多留一个，能用大白话说清就别用成语，大白话里藏着具体，成语里只有态度。

不下判断的开头：
- 值得注意的是 / 需要指出的是 / 重要的是
- 不可否认 / 毋庸置疑
- 众所周知（之后跟着一个并非众所周知的事）

同义词循环（怕重复就换着叫）：
AI 怕重复，同一个东西换着名字叫："该模型……这套系统……这个助手……该引擎……"其实都指一个东西。人写东西不怕重复：认准最清楚的那个词，就一直用它。重复读起来是精确，不是词穷。

修复方法：替换成具体的动作、对象、数字、后果。如果替换不出来，那句话本来就没说什么，删掉。

## 句式层：搭好就空的架子

"在……方面"开头：
- 弱：在用户体验方面，我们做了优化。
- 强：登录页加载从 3 秒降到 0.8 秒。

"通过……，从而……"：
- 弱：通过引入新架构，从而提升了系统稳定性。
- 强：换了新架构后，过去半年没再出过事故。

"不仅……而且……"过度排比：
- 弱：这套方案不仅提升了效率，而且降低了成本，同时还增强了用户体验。
- 强：这套方案省了一半人力，账单也少了三成。

三连排比（中文 AI 文的招牌）：
- 弱：更智能、更高效、更人性化。
- 弱：让数据流转更顺畅、让协作更紧密、让决策更精准。

这种"三短句平行结构"几乎专属于生成式写作。人写文章不会一段塞两组。读到就删。

定语套娃：
- 弱：基于深度学习的面向中小企业的智能化的客户管理解决方案。
- 强：给中小企业用的客服系统，用大模型识别客户意图。

"是……的……"判断句堆叠：
- 弱：这是一次重要的尝试，是我们对未来方向的一次重新思考。
- 强：这次改的是产品定位，不是功能。

倒装"不是 X，而是 Y"：
本身不坏，但 AI 喜欢成段成段地用。一篇文章出现两次以上就要警惕。

时间对比"以前……现在……"：
和"不是 X 而是 Y"一样，是 AI 爱用的对比骨架："以前靠经验，现在靠数据""过去要三天，如今只要三秒"。偶尔一次没问题，成段地用、或全文都靠这个节奏推进，就是机器手笔。改法：直接说现在怎样，把"以前"压成半句或删掉。
- 弱：以前排查线上问题要翻半天日志，现在有了这套工具，几分钟就能定位。
- 强：现在排查线上问题几分钟就能定位，过去得翻半天日志。

回避系动词"是"：
AI 写中文常绕开最朴素的"是"，改用"堪称 / 可谓 / 作为 / 称得上 / 不啻为"来抬调子。
- 弱：这堪称一次重要的尝试，可谓团队的里程碑。
- 强：这是一次重要尝试，对团队是个节点。
能用"是"就用"是"。绕开"是"的句子，十有八九在拗造型。

## 翻译腔层：像机翻的欧化句式

这类句子语法没错，但少了中文母语者的呼吸感，读起来像把英文逐字搬过来。AI 写中文时背后常常是英文语序，所以这味儿特别重。

主语代词过密（中文能省就省）：
- 弱：当你打开应用的时候，它会自动检测你的网络，然后它会为你加载数据。
- 强：打开应用，自动连网、加载数据。

"被"字句滥用（中文多用主动）：
- 弱：这个问题已经被我们很好地解决了。
- 强：这个问题我们修好了。

复数加"们"（中文名词本就不分单复数）：
- 弱：用户们反馈说 / 这些数据们 / 各位开发者们
- 强：用户反馈说 / 这些数据 / 各位开发者

"作为一个……"开头（英文 As a... 的直译）：
- 弱：作为一个做了五年后端的人，我认为这个方案有风险。
- 强：我做了五年后端，这个方案我觉得有风险。

"……之一"（英文 one of the... 的直译，常可删）：
- 弱：这是我们今年最重要的更新之一。
- 强：这是今年最重要的一次更新。（或直接说它为什么重要）

"不仅……而且……" / "一方面……另一方面……"（欧化关联词，AI 成段地用）：
- 弱：这套方案不仅提升了效率，而且降低了成本；一方面解放了人力，另一方面也减少了出错。
- 强：这套方案省了一半人力，账单少了三成，出错也变少了。

"对……进行……" / "使……得到……"（把动词名词化再用空动词激活）：
- 弱：我们对流程进行了优化，使效率得到了提升。
- 强：我们改了流程，跑得更快了。

用身体动作写抽象思考（机翻体感词）：
AI 爱拿身体动作的词去写抽象的思考和系统行为：接住、击穿、拆解、收口、承担、撑不住、不崩、不爆、打穿、收紧、扛住、推开、落地。这些大多是从英文 catch / break / hold up / tighten 直译过来的体感词，中文里本来有更顺的说法。
- 弱：这套设计能接住高并发，不会被流量击穿。
- 强：这套设计扛得住高并发，流量上来也不会宕。
（"扛得住""宕"是中文现成的说法；"接住""击穿"是搬过来的。）

形容词 + 冒号，替读者下判断：
AI 爱先用一个定性的形容词加冒号，把结论塞给读者，再给内容。
- 弱：逻辑很清晰：先建索引，再查询。问题很直接：太慢了。
- 强：先建索引，再查询。慢在每次都全表扫。
删掉定性的形容词，让事实自己说话，清不清晰、直不直接，读者自己会判断。

抽象名词当主语 + 评价性谓语（"X 的 Y 比 Z 更 W"）：
- 弱：工程上的现实比这些数字更难看。
- 强：真跑起来，QPS 只有压测的一半。
把抽象主语换成具体的人、具体的动作，说清到底哪里"不对劲"。

修复方法：把句子读出来。哪里不像中国人平时说话，哪里就是翻译腔。还原成动词、还原成主动、把多余的代词和"们"删掉。yage.ai 那篇文章点得很准，AI 写中文像是"先在英文句法里把意思想清楚，再一个字一个字换成中文"，所以要倒着改：先想中国人会怎么说这件事，再写。

## 结构层：摆出来的章法

"首先 / 其次 / 再次 / 最后"标段头：
中文 AI 文几乎逐段挂这些词。删掉 90%，文章不会断。该过渡的地方让句子本身完成过渡。

"一、二、三"罗马数字伪权威：
不是论文不要用。技术博客挂"一、引言；二、背景"就是套模板。

总分总三段：
开头总结要讲什么 → 中间展开 → 结尾再总结一遍。三遍说同一件事。删掉头尾，保留中间。

【】标记和图标段头：
- 【核心观点】
- 【背景介绍】
- 💡 小贴士
- 👉 重点提示
- 🔥 划重点

中文社区还会出现"敲黑板"、"重点来了"、"上才艺"这种网感堆砌。在严肃文档里全删。

表格滥用：
本该是一段话讲清楚的对比，被强行做成两列表格。表格只在真正需要扫读的对照场景才用。

每段一个二级标题：
英文 AI 文的毛病在中文里更严重。中文段落本身比英文短，加上每段挂 `##`，整篇文章看起来全是骨架没有肉。

## 语气层：演出来的姿态

讨好读者：
- 希望本文对你有所帮助
- 看完别忘了点赞收藏
- 让我们一起……
- 本文将带你……

谦辞套路：
- 抛砖引玉
- 个人浅见
- 不揣冒昧
- 仅供参考

这些没有错，但 AI 用得过于熟练。该有立场的地方用谦辞代替判断，就是稀释。

伪平衡：
- 任何事都有两面性
- 既要……也要……
- 在 A 的同时也要兼顾 B
- 具体问题具体分析

需要分情况讨论时，写出分什么情况、各自结论是什么。"具体问题具体分析"等于没说。结尾也别写那种"什么都对、但什么都没说"的圆滑结论，读完不知道作者站哪边的，等于没写。要么给判断，要么落到具体的下一步。

套路化的开头与收尾：
AI 有几个固定的"起手式"和"收尾式"，模板感极强，看到就换。

- 开头："在这个 XX 的时代，我们每个人都……"（拿宏大背景起手再绕回正题）、"记住，真正重要的是……" / "说到底，关键在于……"（说教式开头，居高临下替读者下结论）。
- 收尾："总之 / 综上所述 / 总而言之 / 归根结底 / 说到底，……"（用一个连接词宣布要总结，然后把前面说过的再说一遍）。

改法：开头直接进入具体的事，别从时代、趋势、"我们每个人"绕；收尾如果只是复述前文就删掉，句子讲完了就停，不用连接词来宣布结束。

强行升华 / 鸡汤收尾：
- 段尾突然拔高到"行业未来"、"时代趋势"、"人类命运"。
- 技术文最后一段变成"这不仅是一次技术升级，更是一次思维方式的革新"。
- 结尾灌鸡汤、喊口号、讲大道理："真正重要的从来不是工具，而是用工具的人。"

砍掉。技术文章不需要史诗结尾，也不需要金句收尾。最后一段删了通常更好，真有话说就落到具体的下一步。

## 中英混用的味道

AI 中文常见的英文残留：
- "做了一个 sanity check"（中文里就说"核对了一遍"）
- "这个 feature 还在 beta"（在产品语境里可保留，在面向普通读者的稿子里换成"还在内测"）
- "上下文（context）"反复双语标注

技术稿可以保留必要的英文术语，但要看读者，给技术读者写就不必每个都中文译；给非技术读者写就别甩英文。

## 标点和排版的痕迹

### 破折号 `——`：现在最扎眼的一个

中文 AI 文最容易被一眼认出的标点，就是破折号。AI 特别爱用它引出转折、补充和判断："这不是小事——而是……""它做一件事——把初稿收尾。"真人也用破折号，但没这么密。**一段里出现两次以上，基本就是机器手笔**，平台和检测工具也最先盯这个。

破折号本身不是错，问题是密度。多数时候按它在干的活换掉就行：

- 引出判断 / 停顿 → 句号或逗号：「这事我想了很久——最后决定不做」→「这事我想了很久，最后决定不做。」
- 补充说明 → 括号：「那套系统——也就是引擎——」→「那套系统（也就是引擎）」
- 总结引出 → 冒号：「它只做一件事——收尾」→「它只做一件事：收尾」
- 实在要保留转折语气，一篇里最多留一两处，且确认那是你自己想要的停顿，不是顺手敲的。

去掉破折号不是"骗检测"，是让文字真的回到人随手打字的样子。靠换标点躲检测既不可靠也偏题；真正的目的是读起来像人写的，少被误判只是顺带的结果。

### 其他机器排版痕迹

- 半角逗号混进中文段（"hello, world，你好"）
- 中文括号里用英文标点（"（参见 chapter 3.）"）
- 全角空格夹在中英文之间（人写一般不加，AI 会自动加）
- 真要用破折号，用中文 `——`，别写成两个减号 `--`；但首选还是少用
- 省略号写成三个点（...）而不是 `……`，且省略号本身也别滥用
- 句子里乱入 `→`、`•`、`·` 这类符号，改成文字或拆句

中文引号用全角 `""` 是正常的，不用动，英文那套"弯引号是 AI 痕迹"的规则不适用于中文。这些单看不是大问题，但成片出现就说明是机器排版。

## 格式形式：换成人最常用的写法

AI 不只爱用 AI 的*符号*，也爱用 AI 的*排版*：满屏加粗、每小段顶一个标题、能一句话说清的偏要列点、段落之间还插分割线和提示框。真人在微信、邮件、文档里几乎不这么排。改法是把每一种"机器排版"换成人随手会打的最朴素形式。

| AI 爱用的形式 | 真人会怎么写 |
|---|---|
| 关键词满屏**加粗** | 不加，让字本身说话；最多一处，往往一处都不加 |
| 每两三句就来个 `##` 小标题 | 少用标题，用段落带过，或干脆不要 |
| 整句整句地列点 | 写成一段话，或句内列举："甲、乙、丙" |
| `1. 2. 3.` 框架 | 用大白话讲；只有真有先后才用"先……再……" |
| 【】段头、👉💡🔥 emoji 引导 | 删掉，直接说事 |
| 三短句排比 | 拆成正常的句子 |
| 段落间的分割线 `---` | 删，空一行就够 |
| "包括以下几点：" 冒号清单 | 直接列在句子里 |
| 「核心要点：」「划重点：」标签 | 把要点直接放进第一句 |
| 两三条信息塞进表格 | 一句话说清 |

一个判断标准：这个格式你会不会打进给朋友的微信里？不会，那多半也不该出现在稿子里。结构只留在读者真要靠它导航的地方：真实步骤、真要查的对照表。

## 改写时的身份与心态

中文 AI 味很多时候不是某个词的问题，是"站位"的问题。改写之前先调整自己的身份设定：

- **别当"资深文案"或"营销专家"。** 一旦进入这个身份，广告词库就自动打开：打造、赋能、极致、丝滑、释放潜能都跟着来。换个身份：发微信的普通朋友、公众号编辑、做了十年的媒体人。同一件事，这几个人会怎么说，就怎么写。
- **先想清楚读者是谁、什么场景。** 发给同事的一条微信，和写给陌生用户的产品说明，写法完全不同。场景模糊时，文字就会退回到谁都不得罪的官腔。具体到人，话才落得下来。
- **多用短句，多用大白话。** 很多 AI 味来自句子太长、修饰词太多。把长句拆短，把名词化的表达还原成动词，少用四字成语，该停顿的地方就停。
- **保留口语里的停顿和语气。** 人说话有节奏、有气口、有犹豫。需要时（人味儿质感模式下）保留语气词和半句话，见 [human-texture.md](./human-texture.md)。

一个最简单的自检：把改完的句子念出来。如果你不会这么跟朋友讲话，就再改一遍。

## 快速识别法

如果一段中文同时满足下面三条以上，基本是 AI 起的：

1. 用了"赋能 / 打通 / 闭环 / 抓手 / 链路"中任何一个
2. 出现三短句排比
3. 有"首先 / 其次 / 最后"或【】段头
4. 段尾上升到"行业 / 时代 / 未来"
5. 全文找不到一个具体数字、人名或场景
6. 一句里堆了两个以上四字成语
7. 满是"被……""作为一个……""不仅……而且……"这类翻译腔句式
8. 破折号 `——` 用得密（一段两次以上），或夹杂弯引号、`…`、`→` 这类机器排版符号

满足越多，重写所需的破坏性越大，往往要拆到只剩观点，重写句子。
