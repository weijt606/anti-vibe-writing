# 中文 AI 味：要删的痕迹

这份参考专门记录中文文档里的生成感。英文那套规则覆盖不到这些——中文有自己一套套话、句式和结构惯性。

按"词汇—句式—结构—语气"四层组织。命中任何一项，基本可以确认是 AI 起的稿。

## 词汇层：套话词

读到这些词，第一反应应该是问："不用这个词，原来要表达什么？"——往往写得出更具体的版本。

互联网黑话：
- 赋能、打通、闭环、抓手、对齐、复盘、心智、链路、底层逻辑、生态位
- 一站式、全方位、全链路、端到端、全场景
- 颗粒度、上下文、落地（被滥用时）

虚动词：
- 助力 / 致力于 / 旨在 / 通过……来……
- 实现 / 完成 / 进行了……（把动作名词化再用"进行"重新激活）
- 具备……能力 / 拥有……特性
- 即可 / 便可 / 仅需

填充词：
- 在很大程度上 / 一定程度上 / 某种意义上
- 进一步 / 不断地 / 持续地
- 更加 / 愈发 / 越来越
- 切实 / 真正 / 实实在在地

夸饰词：
- 干货满满、硬核、深度好文、强烈推荐
- 革命性、颠覆性、划时代、全新升级
- 极致、完美、丝滑、无缝

不下判断的开头：
- 值得注意的是 / 需要指出的是 / 重要的是
- 不可否认 / 毋庸置疑
- 众所周知（之后跟着一个并非众所周知的事）

修复方法：替换成具体的动作、对象、数字、后果。如果替换不出来，那句话本来就没说什么，删掉。

## 句式层：搭好就空的架子

"在……方面"开头：
- 弱：在用户体验方面，我们做了优化。
- 强：登录页加载从 3 秒降到 0.8 秒。

"通过……，从而……"：
- 弱：通过引入新架构，从而提升了系统稳定性。
- 强：换了新架构后，过去半年没再出过事故。

"不仅……而且……"过度排比：
- 弱：这套方案不仅提升了效率，而且降低了成本，同时还增强了用户体验。
- 强：这套方案省了一半人力，账单也少了三成。

三连排比（中文 AI 文的招牌）：
- 弱：更智能、更高效、更人性化。
- 弱：让数据流转更顺畅、让协作更紧密、让决策更精准。

这种"三短句平行结构"几乎专属于生成式写作。人写文章不会一段塞两组。读到就删。

定语套娃：
- 弱：基于深度学习的面向中小企业的智能化的客户管理解决方案。
- 强：给中小企业用的客服系统，用大模型识别客户意图。

"是……的……"判断句堆叠：
- 弱：这是一次重要的尝试，是我们对未来方向的一次重新思考。
- 强：这次改的是产品定位，不是功能。

倒装"不是 X，而是 Y"：
本身不坏，但 AI 喜欢成段成段地用。一篇文章出现两次以上就要警惕。

## 结构层：摆出来的章法

"首先 / 其次 / 再次 / 最后"标段头：
中文 AI 文几乎逐段挂这些词。删掉 90%，文章不会断。该过渡的地方让句子本身完成过渡。

"一、二、三"罗马数字伪权威：
不是论文不要用。技术博客挂"一、引言；二、背景"就是套模板。

总分总三段：
开头总结要讲什么 → 中间展开 → 结尾再总结一遍。三遍说同一件事。删掉头尾，保留中间。

【】标记和图标段头：
- 【核心观点】
- 【背景介绍】
- 💡 小贴士
- 👉 重点提示
- 🔥 划重点

中文社区还会出现"敲黑板"、"重点来了"、"上才艺"这种网感堆砌。在严肃文档里全删。

表格滥用：
本该是一段话讲清楚的对比，被强行做成两列表格。表格只在真正需要扫读的对照场景才用。

每段一个二级标题：
英文 AI 文的毛病在中文里更严重。中文段落本身比英文短，加上每段挂 `##`，整篇文章看起来全是骨架没有肉。

## 语气层：演出来的姿态

讨好读者：
- 希望本文对你有所帮助
- 看完别忘了点赞收藏
- 让我们一起……
- 本文将带你……

谦辞套路：
- 抛砖引玉
- 个人浅见
- 不揣冒昧
- 仅供参考

这些没有错，但 AI 用得过于熟练。该有立场的地方用谦辞代替判断，就是稀释。

伪平衡：
- 任何事都有两面性
- 既要……也要……
- 在 A 的同时也要兼顾 B
- 具体问题具体分析

需要分情况讨论时，写出分什么情况、各自结论是什么。"具体问题具体分析"等于没说。

强行升华：
- 段尾突然拔高到"行业未来"、"时代趋势"、"人类命运"。
- 技术文最后一段变成"这不仅是一次技术升级，更是一次思维方式的革新"。

砍掉。技术文章不需要史诗结尾。

## 中英混用的味道

AI 中文常见的英文残留：
- "做了一个 sanity check"（中文里就说"核对了一遍"）
- "这个 feature 还在 beta"（在产品语境里可保留，在面向普通读者的稿子里换成"还在内测"）
- "上下文（context）"反复双语标注

技术稿可以保留必要的英文术语，但要看读者——给技术读者写就不必每个都中文译；给非技术读者写就别甩英文。

## 标点和排版的小痕迹

- 半角逗号混进中文段（"hello, world，你好"）
- 中文括号里用英文标点（"（参见 chapter 3.）"）
- 全角空格夹在中英文之间——人写一般不加，AI 会自动加
- 破折号写成两个减号（--）而不是 `——`
- 省略号写成三个点（...）而不是 `……`

这些不是大问题，但成片出现就说明是机器排版。

## 快速识别法

如果一段中文同时满足下面三条以上，基本是 AI 起的：

1. 用了"赋能 / 打通 / 闭环 / 抓手 / 链路"中任何一个
2. 出现三短句排比
3. 有"首先 / 其次 / 最后"或【】段头
4. 段尾上升到"行业 / 时代 / 未来"
5. 全文找不到一个具体数字、人名或场景

满足越多，重写所需的破坏性越大——往往要拆到只剩观点，重写句子。
