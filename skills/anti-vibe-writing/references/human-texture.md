# 人味儿质感：可选的"不规范"

这是一份可选 reference。默认模式不会主动注入这些痕迹，它针对的是"显得克制、干净、专业"的稿子。但有些场景反过来：稿子已经够干净，干净到读起来还是不像人写的。

干净不是终点。真人写的东西带着一些"不规范"：倒装、语气词、半句话、突然的停顿、非标准标点、自我修正。AI 学会了避开这些，所以越改越像"无菌室文学"。

这份 reference 给你一组可控的工具，**在用户明确要求"再放松一点 / 像博客风 / 像本人写的"时启用**。不是每篇都用。

## 何时启用

适合：
- 创始人 / 个人博客
- 社交媒体（推特长贴、即刻、小红书）
- 产品发布的"为什么我们做这个"段落
- 内部信、致用户信、致团队信
- 播客 show notes 或脱口秀稿

不适合：
- API 文档 / SDK 说明 / 错误码表
- 合同、隐私政策、监管材料
- 任何需要"权威感"的官方公告
- 学术、技术规范、安全报告

模糊地带（README / 产品文档）：默认不启用。除非用户明说"放松点"。

## 真人写作的 7 类质感

下面列出可调用的"工具"。一次只用一两种，过量会变做作。

### 1. 倒装：把判断推到句首

把强调点提前，句子结构反常规。

- 平：这个功能我用了半年都觉得好用。
- 人：好用，是真的好用。这功能我用了半年。

- 平：贵是这个产品的主要问题。
- 人：贵。这是它的主要问题。

注意点：倒装的"短句 + 完整句"组合最像人写，全篇倒装就成了文学腔。

### 2. 语气词：在判断的地方加一点犹豫或确信

吧、嘛、啊、呗、哦、啦、呀、嘛，每个都有自己的情绪指向。

- 吧：试探 / 让步。"应该够用了吧。"
- 嘛：说服 / 撒娇。"本来嘛，这事就不难。"
- 呗：随意 / 接受。"不行就改呗。"
- 啊：感叹 / 强调。"这才是重点啊。"
- 哦：提醒 / 漫不经心。"记得早点提交哦。"

注意点：每段最多一个。语气词不是装饰，是情绪的位置。错位的语气词比没有更违和。

### 3. 半句 / 残句：句子可以不完整

不是所有句子都要主谓宾齐全。该停的地方就停。

- 完整：这件事我觉得没那么重要。
- 残句：这事，没那么重要。

- 完整：总而言之，事情就是这样了。
- 残句：总之，就这样。

- 完整：我也说不太清楚为什么，但就是觉得别扭。
- 残句：怎么说呢，就是别扭。

注意点：残句必须配合上下文。孤立的残句像标语。

### 4. 自我修正：写出"换了一个说法"的过程

AI 写文一气呵成，人会改主意。把"改主意"留在稿子里。

- "我觉得（其实不是觉得，是确信）这事不能拖。"
- "听起来挺好的，但仔细想想，好在哪儿其实说不上来。"
- "我们做了三版方案。其实是四版，只是第一版没人愿意承认。"

注意点：自我修正用得好是真诚，用多了是耍宝。一篇里不超过两次。

### 5. 非标准标点

中文标点有规范，但人写东西常常破规范。可控地破。

- 用句号代替逗号制造停顿：
  - 规范：贵，且没必要。
  - 放松：贵。而且没必要。

- 用括号补充，制造"想到就补一句"的口语感：
  - 规范：这个方案我们讨论了很久才定。
  - 放松：这个方案（其实吵了不止一次）我们讨论了很久才定。

- 必要时用省略号表达欲言又止（节制使用）：
  - "本来想多写几句的……算了。"

破折号的特别提醒：破折号 `——` 以前是很顺的"个人语气"信号，但现在 AI 把它用滥了，已经反过来成了最扎眼的 AI 标点痕迹（见 [chinese-patterns-to-remove.md](./chinese-patterns-to-remove.md) 的标点一节）。所以人味儿质感模式下也要克制，能用句号停顿、括号补充就别用破折号，一篇最多一两处，且在会被平台或检测扫描的社交稿（推特、Reddit 等）里尽量不用。真要用就用中文 `——`，别写成 `--`。

其他注意点：省略号是 `……`，不是 `...`。破规范是"打破规范"，不是"用错标点"。

### 6. 口语连接词替代正式连接词

- "因此"→"所以"、"结果"
- "综上所述"→"总之"、"说白了"
- "此外"→"还有"、"对了"
- "然而"→"但是"、"不过"
- "首先 / 其次"→"先 / 然后"，或者干脆不写

更口语的：反正、其实、说真的、讲道理、我就这么说吧、咱、咱们

### 7. 具体到怪异的细节

真人写东西会带不必要的具体。AI 写东西会保持得体的抽象。

- 抽象：上次调试花了不少时间，最后发现是个小问题。
- 具体：上次那个 bug 我查了俩小时。一个分号。就一个分号。

- 抽象：用户反馈很积极。
- 具体：有个用户留言说"看到第三段我点了订阅"。第三段。我自己都不知道哪段最关键。

注意点：这种细节不能编。如果原稿里没有，不要凭空加。但如果原稿有材料，把材料从抽象层拉到具体层。

## 随手打（社交专用，最克制的一档）

前面 7 类是"不规范但没错"。这一档不一样：它允许极少量"真人在手机上随手打"的不完美，所以门槛最高，默认不开，只在休闲社交场景、用户明确要开时才启用（怎么算"明确"见下）。

理由很窄：推特、微博、即刻上，真人快打的字本来就不收口，句末懒得点标点、不切大写、漏个"的"、两句连成一句。这种随手恰恰是真人痕迹。但它离"出错"只有一线，所以规矩要死。

### 怎么触发（说清楚就行，不认死某一句）

按意图匹配，不是某句固定咒语。下面任意一种都算明确请求：

- 直接点名这个档：「开随手打 / 启用随手打 / 随手打模式 / casual typing」
- 描述想要的效果：「像手机随手发的 / 像顺手发的 / 别太工整 / 弄随意点，像真人发的 / type it like a quick phone post」

注意：光说"放松一点 / 博客风"只开人味儿质感的常规档，**不**自动开随手打，随手打更激进，需要更明确的信号。拿不准时按"没开"处理，可以反问一句"要不要顺带加点手机随手发的随意感？"

### 能做什么（只省略，不出错）

- 句末该有的句号 / 感叹号，偶尔不点
- 英文不切大写：「ok 那就这样」而不是「OK，那就这样」
- 漏掉可有可无的虚词：「这事我觉得还行」→「这事我觉得行」
- 两个短句不打标点连下来：「试了下 没问题」
- 全角/半角偶尔不讲究

英文场景的对应：全小写、句末不收尾、用 & 代 and、省掉非必要逗号、dont/its 这类口语缩写，同样只在休闲社交。

### 绝对不能做（红线）

- **不碰数字、名称、代码、链接、专有名词、术语、价格、日期**，这些地方一个字都不能差
- **不做会改变意思的错别字**：的/得/地、在/再、哪/那 这类，错了读者会读岔，禁止
- **不在文档、报告、技术稿、公告、落地页里用**，一个错字就掉信任
- 不是"制造错误骗 AI 检测"。目的是手机口语手感，不是躲检测，靠这个躲检测既不可靠也偏题。

### 频率与声明

- 一条短贴最多 1–2 处，长一点也别超过 2–3 处。多了就成了"装"。
- 每次用了，在输出末尾说一句："加了 N 处随手感（漏标点 / 不大写），都在无关紧要处，没碰数字和名称"，给用户一眼回退的机会。

判断是否过量：如果某处"不完美"让读者要回头读一遍才懂，就是过了，改回去。真人随手打也是为了让人看懂，不是为了显得随手。

## 风险与边界

**过量痕迹 = 装真人。** AI 学会"加语气词"之后写出的"嘛、呗、哦"反而成了新的 AI 味。判断是否过量的方法：

- 一段里超过一个语气词 → 删一个
- 一段里两个以上残句 → 改回一个完整句
- 一段里两次以上自我修正 → 砍一次
- 通篇都是"反正 / 说白了 / 其实" → 这不是放松，是油腻

**不要替原作者编情绪。** 加倒装、加语气词、加自我修正，前提是原稿已经表达了那个情绪。你只是把它显化。如果原稿没有，不要凭空注入。

**保留专业感的部分。** 即使是博客风的稿子，里面涉及具体数字、技术细节、产品参数的句子要保持准确，这些地方不放松，反而是真人感的来源之一（人不会在重要数据上撒娇）。

## 启用方式

用户可以这样说，就启用本 reference：

- "放松一点 / loose / 博客风 / 私人语气"
- "像本人写的 / 像创始人自己写的"
- "可以有一点不规范"
- "加点人味"

模型回应时，可以在最末加一句说明："已启用人味儿质感模式，注入了 X 处倒装、Y 处语气词"，给用户回退的机会。

默认情况下不启用。
