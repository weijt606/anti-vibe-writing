# 学习模式：从样本到 host profile

默认的"反 AI 味"是减法，把生成感的痕迹拿掉，留下干净版。但干净不是终点。每个写作者有自己的节奏、用词、停顿习惯。学习模式负责把这些信号从用户的真实样本里提取出来，写成一份可复用的 host profile，让后续的重写更像"那个人写的"，而不是"一个干净版本"。

## 何时启用

学习模式不是默认行为。下面几种情况启用：

- 用户提供了自己写的样本（"参考我之前发的这几篇"、"我推特一般这么写"）
- 用户问"能不能学我的风格"
- 用户已经做了几轮重写后明确说"还是不像我写的"
- 用户准备做系列内容（公众号 / Newsletter / 播客 show notes），想保持风格一致

## 学习的流程

### 第 1 步：收样本

样本越具体越好。问用户提供：

- 至少 3 篇，最好 5–10 篇
- 同类场景的（要学微博风就提供微博样本，要学技术博客就提供技术博客样本）
- 用户本人写的、未经 AI 润色过的
- 最近半年内的（更接近当前风格）

如果用户只能提供少量样本，提取出的 profile 标记为"低置信度"，重写时偏保守。

### 第 2 步：提取风格信号

不要"凭感觉"提风格。按下面这套维度系统提取，每一项都用样本里的具体证据支撑：

**句长分布**
- 平均字数 / 句
- 短句（< 15 字）和长句（> 35 字）的比例
- 是否有"短/短/长"或"长/短"的节奏惯例

**段落结构**
- 平均段长（句数 / 段）
- 习惯单句成段还是多句段
- 是否有"开头一句重击 + 展开"或"流水账"模式

**用词层**
- 高频实词（动词、名词），剔除通用词后留下的偏好
- 是否爱用某些独特说法（口头禅、自造词、行话）
- 第一人称偏好（"我" / "咱" / "本人" / 不写主语）

**句式习惯**
- 是否常用倒装
- 是否常用反问、设问
- 是否常用破折号 / 括号补充
- 是否爱用"不是 X，而是 Y"这类句式

**语气词与情绪标记**
- 用哪些语气词（吧、嘛、啊、呗、哦）
- 频率（每段 / 每篇）
- 是否爱用感叹号 / 省略号 / 三点

**标点节奏**
- 句号 vs 逗号的比例（句号多 → 节奏快 / 干脆）
- 破折号、省略号、感叹号的密度
- 是否有非标准用法（如用句号代替逗号制造停顿）

**话题切入方式**
- 习惯怎么开头：场景描述 / 直给观点 / 提问 / 反常识 / 数据
- 习惯怎么收尾：留钩子 / 升华 / 落到具体行动 / 半句话停住

**禁忌词 / 不会出现的词**
- 样本里完全不出现的"AI 高频词"，比如某人从不说"赋能"
- 这一项很重要：后续重写时这些词要主动避开

### 第 3 步：写 profile

用 `assets/host-profile-template.md` 写一份。模板里每个字段都填，并且**每个字段都附 1–2 句样本原文做证据**，没有证据就不要写进 profile，宁可留空。

### 第 4 步：场景预设叠加

profile 是"这个人怎么写"，预设是"这个场合该怎么写"。两者叠加，规则按下面优先级：

1. host profile 的"禁忌词"和"高频独有词"硬约束，任何场景下都尊重
2. 场景预设决定长度、结构、是否允许 emoji / hashtag
3. host profile 决定句式节奏、语气词、标点习惯

例：一个习惯写长博客的作者写推特发帖时，长度、单段限制服从推特预设，但用词和语气服从他的 profile，不会因为切到推特就突然变成完全不同的人。

### 第 5 步：反馈循环

每次按 profile 输出后，留出回退口：

- "已按你的 profile 重写，重点用了 X / Y / Z 三个特征"
- 问用户："这版像不像？哪里还不像？"
- 用户反馈后，更新 profile（增加 / 删除 / 调整某项特征）

profile 是活文件，不是一次性产物。

## 提取时的常见错误

**过度概括。** "用户喜欢短句"是没用的描述。多短？什么场合短？短句出现在段首还是段尾？要具体到"开头常用 8–12 字短句作为论点钩子"。

**把内容当风格。** 用户样本谈的是 AI，不代表他的风格是"喜欢谈 AI"。剥掉话题，看节奏、用词、句式。

**学到偶发特征。** 5 篇样本里有 1 篇用了感叹号，不代表他爱用感叹号。出现 3 次以上才算特征。

**忽略否定特征。** 样本里**没有**的东西常常比有的东西更标志性。"从不用 emoji"、"从不写赋能"、"从不分小标题"，这些都是硬约束。

**抹平差异。** 同一个人写推特和写博客本来就不一样。如果样本混了多种场景，要分别提取，不要折中。

## 六种场景的学习要点

具体的场景预设参见 [scenario-presets.md](./scenario-presets.md)。学习模式下，每个场景关注的特征不一样：

| 场景 | 学习重点 |
|---|---|
| 推特 / X.com | 单句钩子、留白、是否爱串多条 |
| 微博 / 即刻 / 小红书 | 段落破碎度、emoji / 颜文字偏好、话题标签习惯 |
| 博客 / 公众号 | 开头模式、举例方式、收尾习惯 |
| 播客 show notes | 是否模拟口语、是否分段标 timestamp、是否爱补充上下文 |
| 专业报告 / 内部备忘 | 论证结构、术语密度、对不确定性的表达方式 |
| Reddit / 论坛评论 | 真实细节的具体度、口语连接词（so / honestly）、披露与边界意识 |

## 安全与边界

**不要替用户编风格。** 如果样本不够，明说："我看到的样本只够提取出 X 和 Y 两项特征，其他维度先空着。"不要编。

**不要把风格当永久承诺。** 人会变。每隔一段时间提示用户："这是基于 X 月 Y 日的样本提取的 profile，要不要重新校准？"

**不模仿到失真。** 学习的是风格，不是字面表达。如果原作者样本里出现某句口头禅，不代表新稿子也要塞那句口头禅，除非它本来就该出现。
