# 风格提取提示词

把这段直接交给一个模型，配上用户的样本，可以一次性产出一份 host profile 草稿。然后由人工或下一轮对话校准。

---

## 提示词

你将从一组样本中提取一位作者的写作风格。目标是产出一份可复用的 host profile，让后续重写能更像这位作者，而不是更像"一个干净版本"。

### 输入

下面是用户提供的样本：

```
[在这里粘贴 3–10 篇样本，每篇之间用 --- 分隔]
```

样本类型：[推特 / 微博 / 即刻 / 公众号 / 博客 / 播客 show notes / 专业报告 / 内部备忘 / 其他]

样本数量：[N 篇]

### 任务

按下面 10 个维度提取风格特征。**每一项必须用样本里的具体证据支撑**，证据不足就写"样本不足以判断"，不要编。

1. **句长分布**：平均句长、短长句比例、是否有节奏惯例（如"短/短/长"）
2. **段落结构**：平均段长、是否爱单句成段、典型段落模式
3. **用词偏好**：第一人称用法、高频独有词、口头禅、行业用语、**完全不出现的"AI 高频词"**
4. **句式习惯**：是否常用倒装、反问、破折号补充、残句、"不是 X 而是 Y"
5. **语气词**：用哪些、频率、位置偏好
6. **标点节奏**：句号 / 逗号偏好、破折号 / 省略号 / 感叹号密度、非标准用法
7. **开头惯例**：场景 / 直给观点 / 问题 / 反常识 / 数据 / 其他
8. **收尾惯例**：留钩子 / 升华 / 具体行动 / 半句话停住 / 其他
9. **emoji 使用**：用 / 不用、常用哪些、位置
10. **不确定性表达**：量化 / 直接 / 保守 / 回避

### 输出格式

按下面格式输出，每一项配证据。

```
## 元信息
- 样本数量：N
- 样本类型：[类型]
- 置信度：[低 / 中 / 高]，理由：[说明]

## 1. 句长分布
- 平均：X 字 / 句
- 短句占比：X%
- 长句占比：X%
- 节奏惯例：[描述]
- 证据：[粘 1–2 句]

[... 依此类推，10 个维度都填 ...]

## 提取小结
- 最显著的 3 个特征：[列出]
- 最值得避开的 3 个词 / 句式：[列出]
- 一句话风格画像：[一句话]
```

### 提取原则

- 不要"凭感觉"，每个判断都要有证据
- 不要把内容当风格，剥掉话题本身，看节奏、用词、结构
- 不要学到偶发特征，出现 3 次以上才算惯例
- 重视"否定特征"，样本里完全不出现的常见词，往往比频繁出现的词更标志性
- 不要抹平差异，如果样本是混合场景，分场景列特征，不要折中

### 输出长度

一份 profile 草稿在 800–1500 字之间合适。太短证据不足，太长稀释重点。

---

## 调用示例

```
用户：这是我最近发的 6 条推特和 3 篇博客，能不能学我的风格？
[粘贴样本]

模型：[按上面格式输出 profile 草稿]

模型补一句：profile 草稿已生成。其中 X 项基于 6 条推特，Y 项基于 3 篇博客。
你要不要先看草稿确认，再让我据此重写下一篇？
```

---

## 注意事项

- 如果样本少于 3 篇，不要硬提取，明告用户置信度不足
- 如果样本来自不同场景，建议分别提取两份 profile 再合并
- profile 不要锁死，留下"用户反馈后可更新"的入口
